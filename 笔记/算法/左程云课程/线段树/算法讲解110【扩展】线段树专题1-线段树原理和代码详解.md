如果区间长度是2的次方，树深度为 $log_2 n+1$

如果不是，深度为$log_2 n+2$

准备四倍空间

## 懒标记

以下是对操作的模拟

---

![](../assets/Pasted%20image%2020240727164000.png)

构建如图的线段树，存储1~8的信息，初始设置都为0

对于sum和add

```
sum 1  2  3  4  5  6  7  8
    0  0  0  0  0  0  0  0
add 1  2  3  4  5  6  7  8
    0  0  0  0  0  0  0  0
```

假定执行 $add(1,8,5)$，相当于对1~8下发任务

```
sum 1  2  3  4  5  6  7  8
    40 0  0  0  0  0  0  0
add 1  2  3  4  5  6  7  8
    5  0  0  0  0  0  0  0
```

先“懒”住5，再根据区间长度求当前的sum

---
假定继续执行 $add(5,6,3)$

1. 先将原本的add[1]向下发一层变为

```
sum 1  2  3  4  5  6  7  8
    40 20 20 0  0  0  0  0
add 1  2  3  4  5  6  7  8
    0  5  5  0  0  0  0  0
```

2. 然后任务向右边执行到5~8，3位置

由于任务区间没有把5~8全覆盖，那么继续向下传任务

```
sum 1  2  3  4  5  6  7  8
    40 20 20 0  0  10 10 0
add 1  2  3  4  5  6  7  8
    0  5  0  0  0  5  5  0
```

3. 将任务下发到5~6，6位置

此时5~6完全被任务区间覆盖，所以懒标记，然后返回

```
sum 1  2  3  4  5  6    7  8
    40 20 20 0  0  10+6 10 0
add 1  2  3  4  5  6    7  8
    0  5  0  0  0  5+3  5  0
```

由于对6进行修，上层的值响应的变更

```
sum 1  2  3  4  5  6  7  8
    46 20 26 0  0  16 10 0
add 1  2  3  4  5  6  7  8
    0  5  0  0  0  8  5  0
```

---
继续执行 $add(2,6,-2)$
![](../assets/Pasted%20image%2020240727164000.png)
可以依次写出所有访问的位置
              1~8,1
          1~4,2             5~8,3
     1~2,4    3~4,5      5~6,6    7~8,7
  1~1,8    2~2,9

1. 在2位置将懒标记下传一次
```
sum 1  2  3  4  5  6  7  8
    46 20 26 10 10 16 10 0
add 1  2  3  4  5  6  7  8
    0  0  0  5  5  8  5  0
```

2. 在4位置将懒标记下传一次
```
sum 1  2  3  4  5  6  7  8  9
    46 20 26 10 10 16 10 5  5
add 1  2  3  4  5  6  7  8  8
    0  0  0  5  5  8  5  5  5
```
3. 在9位置，由于2~2被任务区间完全包裹，懒住任务并修改sum
```
sum 1  2  3  4  5  6  7  8  9
    46 20 26 10 10 16 10 5  3
add 1  2  3  4  5  6  7  8  8
    0  0  0  5  5  8  5  5  3
```
4. 修改完3后上层变化
```
sum 1  2  3  4 5  6  7  8  9
    44 18 26 8 10 16 10 5  3
add 1  2  3  4 5  6  7  8  8
    0  0  0  5 5  8  5  5  3
```

5. 然后将目光转回1步骤，1步骤往右边传任务到5节点
由于3~4,5被区间覆盖，懒住任务
```
sum 1  2  3  4 5 6  7  8  9
    44 18 26 8 6 16 10 5  3
add 1  2  3  4 5 6  7  8  8
    0  0  0  5 3 8  5  5  3
```
然后修改上层
```
sum 1  2  3  4 5 6  7  8  9
    40 14 26 8 6 16 10 5  3
add 1  2  3  4 5 6  7  8  8
    0  0  0  5 3 8  5  5  3
```

6. 对于5~8结点同理，最后达成如下效果
```
sum 1  2  3  4 5 6  7  8  9
    36 14 22 8 6 12 10 5  3
add 1  2  3  4 5 6  7  8  8
    0  0  0  5 3 6  5  5  3
```
![](../assets/Pasted%20image%2020240727164000.png)

对于上述操作，以写出数组元素：
$add(1,8,5):$
$[5,5,5,5,5,5,5,5]$
$add(5,6,3):$
$[5,5,5,5,8,8,5,5]$
$add(2,6,-2)$
$[5,3,3,3,6,6,5,5]$
累加为36，印证了上述操作的合理性
